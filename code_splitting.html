✂️ 代码分块

Parcel 支持零配置实现代码分块。 这使你可以将应用程序代码拆分为按需加载的独立包，这意味着更小的初始包大小和更快的加载时间。当用户在应用程序中浏览并需要某个模块时，Parcel 会自动负责按需加载子包。

代码分块是通过动态 import() 方法 syntax proposal 控制的, 工作起来就像 import 语句 or require 方法，不过它是返回一个 Promise，这意味着该模块是异步加载的。

以下示例显示如何使用动态导入来按需加载应用程序的子页面。

import('./pages/about').then(function (page) {
  // 渲染页面
  page.default();
});

由于 import() 返回一个 Promise，所以你也可以使用 async/await 语法。你可能需要配置 Babel 来转换这个语法，直到浏览器得到更广泛的支持。

const page = await import('./pages/about');
// 渲染页面
page.default();

在 Parcel 中动态导入也会延迟加载，所以你仍然可以将所有的 import() 调用放在文件的顶部，并且在使用子包之前，它们将不会被加载。 以下示例显示如何动态地延迟加载应用程序的子页面。

// 设置一个映射，页面名称对应动态 import
// 这些页面在使用前不会加载
const pages = {
  about: import('./pages/about'),
  blog: import('./pages/blog')
};

async function renderPage(name) {
  // 延迟加载请求的页面
  const page = await pages[name];
  return page.default();
}

注意: 如果你想在不支持 async/await 的浏览器中使用它，记得在你的应用中引入 babel-polyfill 或者使用 babel-runtime +babel-plugin-transform-runtime）。

yarn add babel-polyfill
import "babel-polyfill";
import "./app";

阅读文档 babel-polyfill 和 babel-runtime.
