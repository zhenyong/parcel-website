📝 资源类型

正如 资源章节 里描述的，Parcel 将输入的文件看作 资源（Asset）。资源类型 看作是子类，继承自基类 Asset，并实现了必要的接口，用于解析代码、分析依赖、转换以及生成代码。

因为 Parcel 在多处理器核心并行处理资源，因此资源类型可以执行的转换仅限于一次对单个文件进行操作的。对于跨多个文件进行转换，可以使用自定义的Packager。

资源类型接口

const {Asset} = require('parcel-bundler');

class MyAsset extends Asset {
  type = 'foo'; // set the main output type.

  parse(code) {
    // 解析代码，生成 AST 语法树
    return ast;
  }

  pretransform() {
    // 可选，在收集依赖之前执行。
  }

  collectDependencies() {
    // 分析依赖
    this.addDependency('my-dep');
  }

  transform() {
    // 可选，在收集依赖之后执行。
  }

  generate() {
    // 代码生成，你可以返回多个转换（renditions）。
    // 这个结果会传给合适的 packager,用来生成最终的代码包
    return {
      foo: 'my stuff here', // main output
      js: 'some javascript' // alternative rendition to be placed in JS bundle if needed
    };
  }
}

注册资源类型

你可以通过 bundler 的 addAssetType 方法来注册你的资源类型。该方法参数为文件扩展名和资源类型模块的文件路径，这里直接使用文件路径而不是实际的对象，以便传递给 worker 进程。

const Bundler = require('parcel-bundler');

let bundler = new Bundler('input.js');
bundler.addAssetType('.ext', require.resolve('./MyAsset'));
