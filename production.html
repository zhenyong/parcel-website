✨ 生产

当需要打包应用上线生产环境时，可以使用 Parcel 的生产模式。

parcel build entry.js

要更改输出包的目录，可以传 --out-dir [目录]（默认为 dist）

parcel build entry.js --out-dir build/output
# 或缩写 -d ：
parcel build entry.js -d build/output

这将禁用侦测模式和热模块替换，所以它只会构建一次。它还使用压缩器为所有输出包减少文件大小。Parcel 使用的压缩器有：JavaScript 的 uglify-es，CSS 的 cssnano，HTML 的 htmlnano。

启用生产模式还会设置 NODE_ENV = production 环境变量。 像 React 这样的大型库只有开发阶段使用调试功能，通过设置这个环境变量来禁用调试功能，从而使生产的构建更小更快。
